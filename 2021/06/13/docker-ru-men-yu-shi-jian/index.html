一、概念

什么是docker？

Docker是一个开源的引擎，可以轻松的为任何应用创建一个轻量级的、可移植的、自给自足的容器。开发者在笔记本上编译测试通过的容器可以批量地在生产环境中部署，包括VMs（虚拟机）、bare metal、OpenStack 集群和其他的基础应用平台。

三个基本概念：

  • Dokcer镜像（Image） docker镜像类似于虚拟机镜像，可以理解为一个只读模板。镜像是创建docker容器的基础。
  • Docker容器（Container） docker容器类似于一个轻量级的沙箱，docker利用容器来运行和隔离应用。容器是从镜像创建的应用运行实例。
  • Docker仓库（Repository） docker仓库类似于代码仓库，是docker几种存放镜像文件的场所。

应用场景：

  • web应用的自动化打包和发布；
  • 自动化测试和持续集成、发布；
  • 在服务型环境中部署和调整数据库或其他的后台应用；
  • 从头编译或者扩展现有的OpenShift或Cloud Foundry平台来搭建自己的PaaS环境。

二、Docker安装

官网：https://docs.docker.com/engine/install/centos 这里是centos的安装

1、删除旧版本

 sudo yum remove docker \
                  docker-client \
                  docker-client-latest \
                  docker-common \
                  docker-latest \
                  docker-latest-logrotate \
                  docker-logrotate \
                  docker-engine
# 没有安装过的话，会出现下面的信息，忽略进入下一步即可                  
已加载插件：fastestmirror
参数 docker 没有匹配
参数 docker-client 没有匹配
参数 docker-client-latest 没有匹配
参数 docker-common 没有匹配
参数 docker-latest 没有匹配
参数 docker-latest-logrotate 没有匹配
参数 docker-logrotate 没有匹配
参数 docker-engine 没有匹配
不删除任何软件包

2、安装工具包并设置存储库

sudo yum install -y yum-utils

sudo yum-config-manager \
   --add-repo \
   https://download.docker.com/linux/centos/docker-ce.repo

3、安装Docker引擎

sudo yum install docker-ce docker-ce-cli containerd.io

4、启动Docker

sudo systemctl start docker

5、检查

# 检查docker版本
docker -v
# 查看docker已有镜像
sudo docker images
# 设置docker开机启动
sudo systemctl enable docker

5、配置阿里镜像加速器

# 创建目录
sudo mkdir -p /etc/docker
# 配置镜像加速地址 复制到EOF结束
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://u1wa28wg.mirror.aliyuncs.com"]
}
EOF
# 重启后台守护线程
sudo systemctl daemon-reload
# 重启docker
sudo systemctl restart docker

2、镜像命令

Docker Hub：https://registry.hub.docker.com 类似maven云仓库

在之前的介绍中，我们知道镜像是 Docker 的三大组件之一。
Docker 运行容器前需要本地存在对应的镜像，如果镜像不存在本地，Docker 会从镜像仓库下载（默认是 Docker Hub 公共注册服务器中的仓库）。

列出本地主机上的镜像
docker images
REPOSITORY    TAG       IMAGE ID       CREATED        SIZE
mysql         5.7       2c9028880e58   5 weeks ago    447MB
hello-world   latest    d1165f221234   3 months ago   13.3kB
# 显示结果的各个选项说明:
# REPOSITORY：表示镜像的仓库源
# TAG：镜像的标签
# IMAGE ID：镜像ID
# CREATED：镜像创建时间
# SIZE：镜像大小
docker images [options] 
option -a :列出本地所有的镜像（含中间映像层）
       -q :只显示镜像ID
       --digests :显示镜像的摘要信息
       --no-trunc :显示完整的镜像信息
搜索镜像
# 在镜像云仓库搜索镜像 也可以到网站搜索：https://hub.docker.com
docker search 镜像名字 # 例：docker search mysql
docker search [options] 镜像名字
option -filter=3000 : 搜索出来的是收藏数大于3000的。
	   --no-trunc : 显示完整的镜像描述
下载镜像
docker pull 镜像名字 # 例：docker pull mysql:5.7
docker pull 镜像名字[:TAG] # 带镜像标签精准下载，不写tag默认为latest
删除镜像
# 删除镜像 -f强制删除，不管是否是启动状态
docker rmi -f 镜像ID
# 删除多个镜像
docker rmi -f 镜像名1:TAG 镜像名2:TAG 
# 删除所有镜像
docker rmi -f $(docker images -qa)

3、容器命令

说明：有了镜像才能创建容器，例如下载一个Centos镜像 docker pull centos

新建并启动容器
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
# options 参数说明
--name="容器新名字": 为容器指定一个名称；
-d: 后台运行容器，并返回容器ID，也即启动守护式容器；
-i：以交互模式运行容器，通常与 -t 同时使用；
-t：为容器重新分配一个伪输入终端，通常与 -i 同时使用；
-P: 随机端口映射；# 注意为大写的P
-p: 指定端口映射，有以下四种格式
      ip:hostPort:containerPort
      ip:containerPort 主机端口:容器端口 （常用）
      hostPort:containerPort
      containerPort
#使用镜像centos:latest以交互模式启动一个容器,在容器内执行/bin/shell命令。
docker run -it centos /bin/shell
列出运行中的容器
# 列出正在运行的容器
docker ps 
# 带参数命令
docker ps [OPTIONS]
# OPTIONS说明（常用）：
-a :列出当前所有正在运行的容器+历史上运行过的
-l :显示最近创建的容器。
-n：显示最近n个创建的容器。
-q :静默模式，只显示容器编号。
--no-trunc :不截断输出。
退出容器
exit # 退出容器
Ctrl+P+Q # 退出但是容器在后台运行。类似手机后台挂微信、QQ等软件
启动和停止容器
docker start 容器ID # 启动
docker stop 容器ID # 关闭
docker restart 容器ID # 重启
docker kill 容器ID # 停止
删除容器
docker rm 容器ID
docker rm -f $(docker ps -qa) # 删除所有容器 类似删除镜像可以批量

4、其他常用命令

后台启动容器
# 使用镜像centos:latest以后台模式启动一个容器
docker run -d centos
 
问题：然后docker ps -a 进行查看, 会发现容器已经退出
很重要的要说明的一点: Docker容器后台运行,就必须有一个前台进程.
容器运行的命令如果不是那些一直挂起的命令（比如运行top，tail），就是会自动退出的。
 
这个是docker的机制问题,比如你的web容器,我们以nginx为例，正常情况下,我们配置启动服务只需要启动响应的service即可。例如：service nginx start
但是,这样做,nginx为后台进程模式运行,就导致docker前台没有运行的应用,
这样的容器后台启动后,会立即自杀因为他觉得他没事可做了.
所以，最佳的解决方案是,将你要运行的程序以前台进程的形式运行
查看容器日志
docker logs -f -t --tail 10 容器ID
-t 是加入时间戳
-f 跟随最新的日志打印
--tail 数字 显示最后多少条
查看容器的进程信息
docker top 容器ID
